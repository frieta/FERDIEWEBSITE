EARS: Employee Attendance Record System

A comprehensive PHP + MySQL attendance tracking system for employee and visitor management

Project Overview

A PHP + MySQL attendance system for employee time in/time out with visitor logging, admin management, and a guard workflow. EARS provides a complete solution for tracking employee attendance and visitor access with an intuitive public kiosk interface and powerful admin dashboard.

Key Features

  • Public time station for employee and visitor time logs
  • Auto decision for time in/time out based on current-day log status
  • Visitor support with ID format validation (V9999)
  • Optional full visitor data collection during time in
  • Admin dashboard with visitor metrics and recent logs
  • Complete employee management (CRUD operations)
  • Admin user management with role controls
  • Guard account management (create, activate/deactivate)
  • Attendance filtering and CSV export for both employees and visitors
  • Guard panel to complete missing visitor information
  • Theme toggle support (light/dark modes)

Main Workflows

1) Public Kiosk (/index.php)

  • Enter employee number or visitor ID
  • Visitor IDs must match V + 4 digits (example: V0001)
  • Press Enter or use Time In/Time Out buttons
  • For visitors, if full data mode is enabled, a modal requires:
    • Name, Company, Contact No., Purpose, Contact Person, Visitor Type
  • Live cards and tables show:
    • Employees inside
    • Visitors inside
    • Total visitors today
    • Last 10 employee logs
    • Visitors currently inside

2) Admin Panel (/admin)

  • home.php: Dashboard with configurable settings for visitor data collection and time out cooldown
  • employee.php: Employee CRUD operations
  • attendance.php: Employee attendance table with filters and CSV export
  • visitor_attendance.php: Visitor attendance table with date filters and CSV export
  • users.php: Admin user CRUD
  • guards.php: Guard account CRUD and activation controls

3) Guard Panel (/guard)

  • Login with guard account credentials
  • View visitor logs with incomplete details
  • Complete required visitor information fields
  • Save updated visitor records

Requirements & Setup

System Requirements

  • XAMPP (Apache + MySQL)
  • PHP 7.4+ (PHP 8.x works in most cases)
  • MySQL/MariaDB with database named 'attendance'

Installation Steps

  1. Copy project to: C:\xampp\htdocs\EARS
  2. Start Apache and MySQL in XAMPP Control Panel
  3. Create database named 'attendance'
  4. Import schema:
    • Primary: database/attendance.sql
    • Optional: database/schema_update_checker.sql
  5. Access the application:
    • Public kiosk: http://localhost/EARS/
    • Admin login: http://localhost/EARS/admin/
    • Guard login: http://localhost/EARS/guard/

Database Configuration

Configured in admin/db_connect.php:

  • First attempt: host 'localhost', user 'root', password 'root'
  • Fallback attempt: host 'localhost', user 'root', password ''

If your MySQL credentials are different, update admin/db_connect.php accordingly.

Data Model

Core tables utilized by the system:

  • employee - Employee master data
  • attendance - Employee time logs with multiple log types
  • visitor_attendance - Visitor time logs
  • users - Admin user accounts
  • guards - Guard accounts with activation status
  • app_settings - System configuration (visitor data mode, timeout settings)

Security & Production Notes

Current Development State: This codebase uses patterns that are not production-safe by default.

Recommended Hardening Before Production:

  1. Use password_hash() / password_verify() for password security
  2. Replace raw queries with prepared statements to prevent SQL injection
  3. Add CSRF protection for state-changing requests
  4. Regenerate sessions on login and tighten cookie/session configuration
  5. Add input validation and output escaping consistently

Operational Notes

  • Several pages auto-create/auto-update tables and columns if missing
  • Timezone is set to 'Asia/Manila' across multiple files
  • Visitor Time Out requires prior Time In (same day)
  • Time out cooldown is configurable (default: 3 minutes, range: 1-120 minutes)
  • After visitor Time Out, Time In is blocked for cooldown duration to prevent duplicate reversals

Troubleshooting

Database Connection Failed

  • Verify MySQL is running
  • Confirm database 'attendance' exists
  • Check credentials in admin/db_connect.php

Blank Page or PHP Errors

  • Enable PHP error display in local dev environment
  • Set display_errors=On and error_reporting=E_ALL
  • Check Apache/PHP error logs in XAMPP folder

DMS Sales: Document Management System

A comprehensive PHP + MySQL document management application for the Sales department with advanced revision tracking and archiving

Project Overview

DMS Sales is a role-based document management application designed specifically for the Sales department. It provides complete document lifecycle management including uploads, revision history, archiving workflows, license expiration tracking, and a comprehensive notification system with role-based access controls.

Core Features

  • Role-based authentication (admin, authorized, temp)
  • Document upload with client and document type metadata
  • Complete revision history with per-version view and download
  • Archive and unarchive workflow with active/archived file separation
  • License expiration tracking and daily admin summaries
  • Admin notifications feed with notification status handling
  • Temporary-account document permissions with restricted file access
  • Document search, filtering, and folder organization
  • Excel export functionality for license monitoring
  • Document type and client name management
  • Comprehensive audit trail and access logging

Common Workflows

Upload a New Document

  1. Open Document Drive
  2. Click "Add New File"
  3. Fill out client, title, type, and optional expiration date
  4. Upload and save

Revise an Existing Document

  1. Open Document Drive and click "Add New File"
  2. Use the same title and document type as the existing file
  3. Check the revise option in the upload form
  4. Submit to save a new version while preserving the old one in storage/revisions/

Archive / Unarchive

  • Archive from drive views to move file to storage/archived/
  • Unarchive from archive view to restore file to active storage

Check Expiring Documents

  1. Open "License Expiration Monitoring"
  2. Filter by client if needed
  3. Sort by expiration date or client name
  4. Export to Excel when a report is needed

Review Notifications

  1. Open "Notifications" from the top navigation
  2. Choose a filter: All Notifications, Daily Expiration Summary, Document Revised, or New Access Request
  3. Click "Open" to mark a notification as read and follow its target page

Roles and Access Control

Admin Role

Full access to all features, user management, and notifications. Can manage document types and clients.

Authorized Role

Document operations and permitted management actions. Can upload, revise, and archive documents.

Temp Role

Limited access to explicitly assigned documents only. Restricted file access based on permissions.

Installation & Setup

Requirements

  • PHP 7.4+
  • MySQL / MariaDB
  • XAMPP or similar PHP stack
  • Apache web server

Installation Steps

  1. Place the project at: c:/xampp/htdocs/DMS-Sales
  2. Start Apache and MySQL from XAMPP Control Panel
  3. Create/import database schema:
    • Open phpMyAdmin
    • Import database/schema.sql
  4. Update credentials in: config/database.php
  5. Open the app: http://localhost/DMS-Sales/public/

Storage and File Management

  • Active uploads are stored under storage/active/
  • Archived files are moved to storage/archived/
  • Previous versions are copied to storage/revisions/ when a document is revised
  • File history is tracked in document_revisions table and surfaced in "Revised Files" page
  • Storage folders must be writable by Apache/PHP process

Important Configuration Notes

  • Ensure storage/active, storage/archived, and storage/revisions folders are writable
  • App routes and assets are configured for base path /DMS-Sales/public
  • If you rename the project folder, update hardcoded paths accordingly
  • Login redirects to public/dashboard.php after successful authentication
  • The default landing page at public/index.php redirects to login or dashboard depending on session state

Troubleshooting

Cannot Log In

  • Verify DB credentials in config/database.php
  • Ensure document_management_system database exists
  • Confirm schema import succeeded

Upload or Revision Errors

  • Check storage folder write permissions
  • Verify Apache user has write access to storage directories
  • Ensure sufficient disk space is available

Missing Notifications

  • Confirm your account role allows notifications
  • Verify notification records exist in admin_notifications table
  • Check that notification settings are enabled in admin panel

Wrong Links After Moving Folder

  • Update base path references using /DMS-Sales/public
  • Check configuration files for hardcoded paths
  • Clear browser cache if links still appear incorrect

## ⚙️ System Requirements

- **Server**: Apache 2.4+ with PHP 8.0+
- **Database**: MySQL 5.7+ or MariaDB 10.3+
- **Browser**: Modern browser (Chrome, Firefox, Safari, Edge)
- **Storage**: Minimum 500MB for documents, backups, and reports
- **Internet**: Optional (local network deployment supported)

## 🎯 Core Features

### Financial Document Management
- **DR-SI (Sales Invoices)**: Create, approve, post, track payments
- **OR-CR (Payment Receipts)**: Record payments, allocate to invoices
- **Multi-Attachment System**: Attach files with automatic versioning
- **Revision Tracking**: Complete history of document changes

### Financial Analysis & Reporting
- **Statement of Account (SOA)**: Complete client transaction history
- **Accounts Receivable Aging**: Overdue analysis by days past due
- **Summary Reports**: Revenue and receipt analysis
- **Scheduled Reports**: Automatic generation on configured dates
- **CSV Export**: All reports exportable to Excel

### Client Management
- Complete client master file with contact information
- Client balance tracking and credit management
- Financial history and transaction linking
- Deactivation (instead of deletion) for audit trail

### System Administration
- **User Management**: Create users, assign roles, manage permissions
- **Multi-Level Approvals**: Workflow-based document approval
- **Database Backup & Recovery**: Automatic daily + manual backups
- **Audit Trail**: Complete logging of all system activities
- **Admin Notifications**: Real-time alerts for submissions and actions

### UI, Mobile & PWA Experience
- **Consistent Modals**: Unified modal close controls and behavior across modules
- **Responsive Layout**: Mobile-friendly UI while preserving desktop workflows
- **Install Prompts**: Smart mobile/desktop install prompts that hide after install
- **Mobile Full Screen**: Optional in-app mobile full-screen toggle (browser support dependent)
- **PWA Support**: Manifest + service worker for shortcut-based app launch behavior

### Security & Compliance
- 6-level role-based access control
- bcrypt password hashing
- SQL injection prevention (prepared statements)
- CSRF token protection
- Checksum verification (SHA-256) for backups
- Complete audit trail for compliance
- Self-protection safeguards for admin accounts

## 💾 Database Features

### Core Tables
- **users**: System user accounts and roles
- **client_names**: Client master file
- **drsi_records**: Sales invoice records
- **orcr_records**: Payment receipt records
- **documents**: Document metadata and storage
- **financial_attachments**: Multi-file version tracking
- **audit_trail**: Complete activity log
- **admin_notifications**: Alert system

### Automatic Features
- Timestamp tracking (created_at, updated_at)
- User attribution (created_by, modified_by)
- Referential integrity (foreign keys)
- Cascade delete for linked records
- Unique constraints for key fields

## 🔄 Workflow Examples

### Create to Payment Flow (30-60 minutes)
1. **Create** DR-SI (Encoder) → Save as draft
2. **Upload** supporting documents
3. **Submit** for approval
4. **Approve** (Admin) → Status changes to approved
5. **Post** (Accountant) → Posted to ledger
6. **Record Payment** → OR-CR created
7. **Allocate** payment to invoice
8. **Post** payment → Balances updated

### Report Generation Flow
1. **Configure** schedule (Admin) → Day of month/week
2. **System generates** automatically on date
3. **Review** reports in dashboard
4. **Download** CSV files
5. **Analyze** in Excel
6. **Distribute** to stakeholders

*Complete workflows: [SYSTEM_DOCUMENTATION.md](SYSTEM_DOCUMENTATION.md) → Complete Workflow Processes*

## 📈 Performance & Optimization

### Recommended Practices
- Archive records after 6 months
- Regular database optimization (monthly)
- Backup verification (weekly)
- User access review (quarterly)
- Storage cleanup (monthly)

### Scaling Guidelines
- <1000 invoices/month: Current setup sufficient
- 1000-10000/month: Consider database indexing optimization
- 10000+/month: Plan for database archiving strategy
